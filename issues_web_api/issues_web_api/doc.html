Mediatypes

This API supports the following media types:

  • application/vnd.collection+json
  • application/vnd.siren+json

Makes use of the media type application/api-problem+json to present a more detailed error message.

Homepage Link Relations

This relations are application-based and are present in the homepage served when the main uri of this API is accessed through a GET request.

comments_url

The target URI is a templated URI that can be used to obtain a certain comment or a collection of comments related to a certain project and to a certain issue.

issues_url

The target URI is a templated URI that can be used to obtain a certain issue or a collection of issues related to a certain project.

projects_url

The target URI is a templated URI that can be used to obtain a certain project or a collection of projects.

tags_url

The target URI is a templated URI that can be used to obtain a certain tag or a collection of tags related to a certain project.

Resources

The following items describe the properties in each resource available through this API.

Project

All projects have the following properties:

Property Description DataType Nullable Unique Readonly
id unique key for project String FALSE TRUE TRUE
tags set of tags related to the project Array of String TRUE FALSE TRUE
name unique name for project String FALSE TRUE TRUE
id is only available after a project is created.

Issue

Property Description DataType Nullable Unique Readonly
id unique key for issue inside a given project String FALSE TRUE TRUE
title title of issue String FALSE FALSE TRUE
state current status of issue String FALSE FALSE FALSE
tags tags associated with an issue Array of String TRUE FALSE FALSE
description the main description of issue String FALSE FALSE FALSE
id is only available after an issue is created.

Tag

Property Description DataType Nullable Unique Readonly
id unique key for a tag String FALSE TRUE TRUE
name name of tag String FALSE FALSE TRUE
id is only available after a tag is created.

Comment

Property Description DataType Nullable Unique Readonly
id unique key for a comment String FALSE TRUE TRUE
content content of comment String FALSE FALSE TRUE
creationDate timestamp when comment was created Date FALSE TRUE TRUE
id and creationDate are only available after a comment is created.
creationDate follows the pattern "MM/dd/yyyy".

Link Relations

This relations are application-based and are present in the representations of each ressource.

issues

The target URI points to a resource that represents a collection with items of the resource type Issue.

issue

The target URI points to a resource that represents a single item of the resource type Issue.

tags

The target URI points to a resource that represents a collection with items of the resource type Tag.

tag

The target URI points to a resource that represents a single item of the resource type Tag.

comments

The target URI points to a resource that represents a collection with items of the resource type Comment.

comment

The target URI points to a resource that represents a single item of the resource type Comment.

project

The target URI points to a resource that represents a single item of the resource type Project.

search

The target URI can be used to restrict the representation of a collection for the context resource.
